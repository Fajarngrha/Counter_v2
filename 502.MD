# Panduan Fix 502 Bad Gateway (Nginx + Node.js)

Dokumen ini jadi SOP cepat jika web dashboard menampilkan **502 Bad Gateway**.

## Arti Error 502

`502 Bad Gateway` berarti:
- Nginx aktif,
- tapi backend (Node.js di port `3010`) tidak bisa diakses.

Penyebab paling umum di project ini:
- service `iot-counter` crash,
- permission `data/db.json` atau `data/shift-config.json` salah (`EACCES`),
- file JSON rusak/ada merge conflict marker (`<<<<<<<`),
- backend belum listen di port `3010`.

---

## 1) Cek Cepat (2 menit)

Jalankan:

```bash
cd /opt/iot-counter/Counter_v2
sudo systemctl status iot-counter --no-pager
sudo journalctl -u iot-counter -n 120 --no-pager
curl -I http://127.0.0.1:3010/api/session
sudo ss -ltnp | rg 3010
```

Interpretasi:
- Jika `curl` ke `127.0.0.1:3010` gagal -> backend tidak jalan / crash.
- Jika backend jalan tapi domain tetap 502 -> cek Nginx (langkah 5).

---

## 2) Fix Paling Sering: Permission Denied (EACCES)

Jika log berisi:
- `EACCES: permission denied, open '/opt/iot-counter/Counter_v2/data/db.json'`

Jalankan:

```bash
cd /opt/iot-counter/Counter_v2
sudo mkdir -p data
sudo touch data/db.json data/shift-config.json

sudo chown -R iot-svr:iot-svr /opt/iot-counter/Counter_v2
sudo chown -R iot-svr:www-data /opt/iot-counter/Counter_v2/data
sudo chmod -R 775 /opt/iot-counter/Counter_v2/data
sudo chmod g+s /opt/iot-counter/Counter_v2/data
```

Validasi:

```bash
sudo -u www-data test -w /opt/iot-counter/Counter_v2/data/db.json && echo "WRITE OK" || echo "WRITE FAIL"
```

Restart:

```bash
sudo systemctl restart iot-counter
```

---

## 3) Fix Jika JSON Rusak / Conflict Marker

Jika log berisi:
- `Unexpected token < in JSON`
- atau terlihat `<<<<<<< Updated upstream`

Langkah:

```bash
cd /opt/iot-counter/Counter_v2
sudo cp data/db.json data/db.json.bak.$(date +%F-%H%M%S)
sudo cp data/shift-config.json data/shift-config.json.bak.$(date +%F-%H%M%S) 2>/dev/null || true

sudo git checkout --ours data/db.json data/shift-config.json
rg "<<<<<<<|=======|>>>>>>>" data/db.json data/shift-config.json
```

Cek JSON valid:

```bash
node -e "JSON.parse(require('fs').readFileSync('data/db.json','utf8')); console.log('db.json OK')"
node -e "JSON.parse(require('fs').readFileSync('data/shift-config.json','utf8')); console.log('shift-config.json OK')"
```

---

## 4) Jika Service Aktif Tapi Tidak Listen Port 3010

Jalankan manual untuk lihat error asli:

```bash
cd /opt/iot-counter/Counter_v2
npm start
```

Biasanya error langsung terlihat jelas (env, path, syntax, permission).

Jika dependency berubah:

```bash
npm install --omit=dev
sudo systemctl restart iot-counter
```

---

## 5) Jika Backend Sehat Tapi Domain Tetap 502

Cek Nginx:

```bash
sudo nginx -t
sudo systemctl reload nginx
sudo tail -n 80 /var/log/nginx/error.log
sudo nginx -T | rg -n "server_name|proxy_pass|3010"
```

Pastikan upstream:
- `proxy_pass http://127.0.0.1:3010;`

Dan websocket header ada:

```nginx
proxy_set_header Upgrade $http_upgrade;
proxy_set_header Connection "upgrade";
```

## 6) Checklist Pulih

Semua ini harus lolos:

```bash
sudo systemctl status iot-counter --no-pager
curl -I http://127.0.0.1:3010/api/session
curl -I http://iotsrv.fcc-id.id
```

Target hasil:
- service `active (running)`
- localhost API merespons
- domain publik tidak 502

---

## 7) Pencegahan Agar 502 Tidak Berulang

1. Gunakan 1 user deploy konsisten (mis. `iot-svr`), hindari campur `sudo npm`.
2. Folder `data/` wajib writable oleh `www-data`.
3. Setelah `git pull`, cek cepat:
   - `node -e "JSON.parse(...db.json...)"`
4. Jika ada stash/merge conflict, selesaikan dulu sebelum restart service.
5. Simpan backup rutin:
   - `data/db.json`
   - `data/shift-config.json`
